# FastTodo

Минималистичный todo-менеджер: задачи с именованными пунктами-чекбоксами, секциями, прогресс-баром и двумя режимами просмотра.

## Возможности

### Задачи и пункты
- **Задачи** — создание, удаление, переименование, сортировка перетаскиванием
- **Пункты** — чекбоксы с произвольными названиями; пустое имя → `Пункт №1`, `Пункт №2`, …
- **Секции** — группировка пунктов с заголовком, счётчиком `выполнено/всего`, сворачиванием; пункты вне секций добавляются формой внизу списка
- **Прогресс** — процент выполнения и анимированный бар (толчок при отметке, confetti при 100%)
- **Две вкладки** — «Текущая» (детальный вид) и «Все задачи» (список карточек)

### Интерфейс
- **Навигация** — переключение между задачами кнопками с названиями соседних задач
- **Темы** — 8 цветовых схем (dark, light, orange-green, sakura, ocean, sunset, neon, forest)
- **Drag-and-drop** — пункты (внутри и между секциями), секции, задачи; placeholder показывает место вставки
- **Текст пунктов** — авто-раскрытие многострочного текста при наведении и редактировании
- **Память** — последняя открытая задача и выбранная тема в `localStorage`; свёрнутые секции тоже запоминаются

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

Откройте [http://127.0.0.1:5000](http://127.0.0.1:5000).

При первом запуске создаётся `todo.db` в корне проекта.

### Параметры

```bash
python app.py --host 0.0.0.0 --port 5000   # доступ из сети
python app.py --debug                       # только с --host 127.0.0.1 (по умолчанию)
```

`--debug` на публичном адресе заблокирован: debug-консоль Werkzeug небезопасна.

## Как это устроено

### База данных

Три таблицы:

- `tasks` — `id`, `title`, `position`, `created_at`
- `items` — `id`, `task_id`, `name`, `completed`, `position`, `section_id` (nullable)
- `sections` — `id`, `task_id`, `title`, `position`

Порядок задач, секций и пунктов хранится в `position`. Пункт может быть вне секций (`section_id = NULL`) или внутри одной секции.

### API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/tasks` | Список задач с пунктами, секциями и прогрессом |
| `POST` | `/api/tasks` | Создать задачу |
| `GET` | `/api/tasks/:id` | Одна задача |
| `PATCH` | `/api/tasks/:id` | Переименовать задачу |
| `DELETE` | `/api/tasks/:id` | Удалить задачу |
| `PUT` | `/api/tasks/reorder` | Порядок задач (`{ "order": [3, 1, 2] }`) |
| `POST` | `/api/tasks/:id/items` | Добавить пункт (`section_id` опционально) |
| `PUT` | `/api/tasks/:id/layout` | Порядок пунктов и секций |
| `POST` | `/api/tasks/:id/sections` | Создать секцию с пункта (`from_item_id`) |
| `POST` | `/api/tasks/:id/sections/ungroup-from` | Вынести пункты из секции |
| `PATCH` | `/api/sections/:id` | Переименовать секцию |
| `DELETE` | `/api/sections/:id` | Удалить секцию |
| `PATCH` | `/api/items/:id` | Отметить / переименовать пункт |
| `DELETE` | `/api/items/:id` | Удалить пункт |

### Frontend

Одностраничное приложение без сборщика. Точка входа — `static/js/main.js` (ES modules).

Глобальное состояние в `core/state.js`:

- `tasks` — кэш с сервера
- `currentTaskId` — выбранная задача (`localStorage`: `fasttodo:lastTaskId`)
- `view` — активная вкладка

При переключении чекбокса состояние обновляется оптимистично — список пунктов не перерисовывается целиком, чтобы не сбивать hover и анимации.

Секции рендерятся блоками с заголовком; layout (порядок пунктов и секций, принадлежность к секции) читается из DOM и сохраняется через `PUT /layout`.
